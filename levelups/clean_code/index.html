Clean Code

(a beginners guide)

What is clean code?

Code that makes things easy

  • Easy to understand
  • Easy to modify/build upon
  • Easy to fix when things go wrong
"You know you are working on clean code when each routine you read turns out to be pretty much what you expected."
Ward Cunningham

"Clean code always looks like it was written by someone who cares."
Michael Feathers

Why bother?

"Programming is the art of telling another human what one wants the computer to do."
Donald Knuth

Clean code isn't for the computers, it's for us

Happy colleagues!

  • Help your future self and everyone else
  • Make work more enjoyable
  • Take pride in what we produce

How do we achieve it?



  • Ongoing process
  • Focus on small things
  • Naming
  • Functions
  • Formatting

Naming



  • Naming things is hard
  • Be expressive
  • Be consistent
  • Functions should be verbs

  • No unecessary crap
  • Use real words
  • Long and descriptive
    better than short
    and cryptic
  • Don't be scared to
    rename anything

Functions




  • As small as possible
  • As few arguments as possible
  • Don't repeat yourself:
    "Duplication may be the root
    of all evil in software"

  • Do one thing
  • No side effects
  • Operate at single level
    of abstraction

Formatting




  • Read top down
  • Things should be where
    you expect them to be
  • Useful comments only
  • Stick to the style guide!
